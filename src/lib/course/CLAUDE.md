# src/lib/course

Scoping steps. One file = one LLM call wrapped up.

- Each lib step: validate course state → run `executeTurn` with `responseSchema` (no translator helpers, no parser closures) → persist the wire shape directly to `courses.{column}` → return typed payload + `nextStage`.
- Untrusted input reaches the prompt envelope via `renderStageEnvelope`, which XML-escapes.
- Routers (`src/server/routers/`) sequence steps; UI never imports from here.
- `submitBaseline` is the scoping-close step: mechanical MC grading first, then one append-only `executeTurn` against `makeScopingCloseSchema` (extends the shared `makeCloseTurnBaseSchema`). `persistScopingClose` then widens the JSONB, upserts concepts at default (untaught) SM-2, opens Wave 1 from `seedSource.scoping_handoff.blueprint`, seeds the assistant `openingText`, flips status to `active`, and bumps `totalXp` — one transaction.
- Wave turns have dual transports over one spine: `prepareWaveTurn` (guards + learner input) and `persistWaveMidTurn`/`executeWaveClose` (persistence) are shared by `submitWaveTurn` (blocking, tRPC `wave.submitTurn` — rollback path) and `streamWaveTurn` (UI message stream, consumed by the route handler under `src/app/api/course/`). Behavior must stay identical; fix the shared pieces, not one transport.
- Streamed mid-turns emit structure via the TOOL channel, not the mega-schema: `streamWaveTurn` runs `executeToolTurnStream` with per-attempt tools from `waveTurnTools.ts` (`buildWaveMidTurnTools` + one `WaveTurnCollector` per attempt) and the post-loop gate `waveMidTurnGate.ts`. Tool executes are STAGING ONLY (collector mutation — no DB, XP, or SM-2); `persistWaveMidTurn` drains the collector unchanged. The blocking transport keeps the mega-schema (`outputContract: "json"`) until its TODO.md deletion.
- Tool input schemas in `waveTurnTools.ts` are deliberately FLAT (wire + validator): `anyOf` unions on the tool wire crater gpt-oss-120b validity ~95%→~44% (probe verdict finding 2, reproduced live). Keep unions out of tool wire bytes; map flat→canonical in the executes. The `superRefine` directive strings are prompt-engineered — never paraphrase.
- Questionnaire secrets (`correct`, `freetextRubric`) must never reach the client pre-submission: `streamWaveTurn` allowlist-redacts forwarded questionnaire tool inputs, drops `tool-input-delta` chunks, AND cuts text-delta forwarding at an unfenced line-start `{` (`findJsonProseLeakIndex`) — a JSON-imitation attempt dumps the answer key into the TEXT channel, which no tool-chunk redaction touches. Preserve all three on any forwarding change.
